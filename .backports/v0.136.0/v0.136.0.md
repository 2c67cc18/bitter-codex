## DAG

Implementation can follow upstream chronology within each bundle. Cross-bundle
dependencies are:

```text
shell-runtime-cleanup                    (independent)

app-server-thread-history-client-ids     (independent)

app-server-archive-resume-guard          (independent)

web-tool-runtime                         (independent)

auth-refresh                             (independent)

dynamic-tool-storage-cleanup             (independent)
```

Parallel lanes:

- Lane A: `shell-runtime-cleanup`
- Lane B: `app-server-thread-history-client-ids`
- Lane C: `app-server-archive-resume-guard`
- Lane D: `web-tool-runtime`
- Lane E: `auth-refresh`
- Lane F: `dynamic-tool-storage-cleanup`

One reasonable serial order:

1. `shell-runtime-cleanup`
2. `app-server-thread-history-client-ids`
3. `app-server-archive-resume-guard`
4. `web-tool-runtime`
5. `auth-refresh`

## Bundles

### Shell Runtime Cleanup

Manual-port with `specs/shell-runtime-cleanup.md`:

- `9152ebd28` - preserve shell cleanup on interruption

The patch touches surviving shell/runtime behavior, but upstream also changes
tool handlers and runtime plumbing that have different local shapes.

### App-Server Thread History / Client IDs

Manual-port with `specs/app-server-thread-history-client-ids.md`:

- `2a1158b8e` - include turns page on thread resume
- `e92c952b2` - add user input client IDs

These are retained app-server v2 and thread-history behaviors. Upstream patch
shape includes TUI, analytics, app-server-test-client, and broad generated
schema churn that should not be copied directly.

### App-Server Archive Resume Guard

Manual-port with `specs/app-server-archive-resume-guard.md`:

- `3e7baa00e` - archived session resume guard from thread archive CLI work

Port only the retained app-server bug fix that rejects resume/fork by id for
archived sessions with an unarchive hint. Do not restore upstream CLI/TUI
archive command plumbing.

### Web Tool Runtime

Manual-port with `specs/web-tool-runtime.md`:

- `96f1347fa` - show activity for standalone web search calls
- `1f93706e9` - require model for standalone web search
- `1c55bb270` - improve built-in tool schema docs

Only the retained standalone web-search request/activity behavior and surviving
tool descriptions should be ported. Do not restore code-mode, extension,
multi-agent, or removed tool-handler docs.

### Auth Refresh

Manual-port with `specs/auth-refresh.md`:

- `6111791d0` - treat `refresh_token_reused` 400s as relogin-required
- `e5afe5bf8` - refresh near-expiry ChatGPT access tokens before requests

The `codex-login` crate survives locally, and both patches map to existing
ChatGPT token refresh behavior.

### Dynamic Tool Storage Cleanup

Manual-port with `specs/dynamic-tool-storage-cleanup.md`:

- `304d15cab` - remove redundant SQLite dynamic tool storage

Dynamic tools and rollout session metadata still exist locally. The port should
remove the duplicate SQLite/thread-store persistence path while preserving
rollout-based resume behavior.

## Defer

- `1c7832ffa` - pending response item storage is useful session behavior, but
  the upstream rewrite depends on removed agent, goal, guardian, hook,
  code-mode, and extension surfaces. Reconsider only with a local session-state
  design.
- `f4e9d2caa` - config document helper extraction is a refactor with no
  standalone behavior change; take only if future config edits need it.

  code-mode, guardian, multi-agent, extensions/plugins, MCP client crates, TUI,
  Python SDK, debug-client, Windows sandbox, or remote-control surfaces.
- Dependency updates are manual ports only when the dependency still exists in
  the local workspace.
