## Bundles

### Thread Title Reconciliation

Manual-port with `specs/thread-title-reconciliation.md`:

- `3cdce52865` - preserve renamed thread titles during reconciliation

The rollout, SQLite state, and thread-store metadata surfaces survive locally,
but the upstream patch sits near adjacent memory-mode and generated-test churn
that differs in this fork. Port only the retained behavior that keeps explicit
SQLite thread titles when reconciliation/backfill observes an inferred rollout
title.

### Startup WebSocket Prewarm

Classification: `cherry-pick`.

- `859dbe2761` - skip startup prewarm when websockets are disabled

This is a small retained `codex-core` startup fix. It avoids building prewarm
prompt/tool state for HTTP-only providers when no reusable WebSocket session
can result.

### Parallel Standalone Web Search

Manual-port with `specs/parallel-standalone-web-search.md`:

- `b3c4157034` - enable parallel standalone web search calls

Standalone web-search and parallel tool execution survive locally, but the
upstream patch is extension-shaped. This fork now carries the behavior through
the native/dynamic tool runtime, so port only the retained behavior by making
the local standalone web-search handler advertise parallel-call support.

### Running Thread Path Normalization

Classification: `manual-port`.

- `ad355d4c96` - Fix Windows running thread resume path normalization

The upstream repro is Windows-specific, but the retained behavior is app-server
resume path comparison, not Windows sandbox or release support. This fork does
not retain Windows sandbox/packaging surfaces, so port this only if the local
app-server is expected to accept Windows-style paths; otherwise keep it
documented as a scoped portability fix rather than restoring any Windows-only
surface. The exact upstream test path differs locally because the app-server v2
suite has been pared down, so add focused local coverage where practical.

### Explicit Rollout Path Validation

Classification: `manual-port`.

- `b89bf1ef47` - Reject directory rollout paths for pathless side chats

The upstream repro is side-chat-shaped, but the retained behavior is broader:
explicit rollout paths should resolve to actual rollout files before
thread-store/app-server resume or fork attempts read them. Port the
thread-store/rollout validation that rejects directories and non-files while
leaving absent side-chat-only test scaffolding behind.

### Remove Persist Extended History

Classification: `manual-port`.

- `11e0f3d3ae` - app-server: remove experimental persist_extended_history bool flag

The retained app-server v2 API still accepts `persistExtendedHistory`, but the
local app-server only emits a deprecation notice and forces
`persist_extended_history: false` before starting core threads. Port the
retained cleanup that removes the disabled request flag and dead core/thread
store plumbing, adapting around local tests and any schema generation
differences.

### Enterprise Rate Limit Projection

Classification: `manual-port`.

- `c8e5db16c9` - feat: show enterprise monthly credit limits in status

The retained value is carrying optional `individualLimit` through the existing
rate-limit snapshot flow in `backend-client`, `codex-protocol`, `codex-api`,
core rate-limit updates, and app-server v2 account/rate-limit APIs. Port only
the retained data/API projection and generated app-server/backend schemas;
leave upstream TUI `/status` rendering excluded.

## Defer

- `a8a6071279`, `4eded02f52`, `01cb97851b`, `917a9a41a3`, `c3cdf3c007`,
  `9f4fac8ec4`, `7c285f9e9c`, `45912a6dc6` - compressed rollout support is
  a new cold-storage representation for retained rollout/thread-store behavior.
  Decide whether this fork supports rollout compression at all before porting;
  the bundle carries dependency, migration, metrics, search, materialization,
  and writer/compressor coordination implications.
- `d6748f741a`, `feb9eddc51`, `009e6c4817` - unified-exec zsh fork gating and
  shell override cleanup touch retained tool planning, but they depend on
  upstream packaged zsh-fork behavior previously not carried wholesale. Decide
  whether this fork supports that runtime before porting.
- `3766941161` - running async main on a sized stack may help CLI stability,
  but the upstream commit notes a `Send` boundary check failure. Reconsider
  with the adjacent upstream fixes that made the release buildable.
- `1dd731305a` - core stack-pressure reductions in session startup may be
  useful and relate to the async-main stack change, but the upstream patch also
  includes TUI config rebuild changes and sits near broader startup refactors.
  Reconsider as a focused core-only stack pass with the async-main stack
  change.
