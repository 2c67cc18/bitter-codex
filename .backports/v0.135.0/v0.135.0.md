# `rust-v0.135.0` Backport Pass

Tag: `rust-v0.135.0`

Policy: `BACKPORT_POLICY.md`

## DAG

Implementation can follow upstream chronology within each bundle. Cross-bundle
dependencies are:

```text
dependency-toolchain ─┐
                      ├─> remote-compaction-v2
turn-metadata-context ┘

runtime-feature-cleanup ──> runtime-selected-web-tool

image-tool-compatibility     (independent)
resume-rollout-history       (independent)
rollout-writer-diagnostics   (independent)
```

Parallel lanes:

- Lane A: `dependency-toolchain` -> `remote-compaction-v2`
- Lane B: `turn-metadata-context` -> `remote-compaction-v2`
- Lane C: `runtime-feature-cleanup` -> `runtime-selected-web-tool`
- Lane D: `rollout-writer-diagnostics`
- Lane E: `image-tool-compatibility`
- Lane F: `resume-rollout-history`

`remote-compaction-v2` waits for both Lane A and Lane B. Other lanes can run in
parallel unless a local conflict appears during implementation.

One reasonable serial order:

1. `dependency-toolchain`
2. `rollout-writer-diagnostics`
3. `image-tool-compatibility`
4. `resume-rollout-history`
5. `turn-metadata-context`
6. `remote-compaction-v2`
7. `runtime-feature-cleanup`
8. `runtime-selected-web-tool`
